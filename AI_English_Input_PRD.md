## 一、产品概述

### 1.1 产品名称
AI English Input

### 1.2 产品一句话描述
一款 Chrome 浏览器插件，让用户在 WhatsApp Web、Gmail 及常见网页输入框中用中文输入，触发后即时生成 3 个英文表达候选，一键填入输入框，由用户自行发送。

### 1.3 核心价值主张
- 用中文思考，用地道英文发送
- 专为跨境电商、外贸销售、海外客服设计
- 不切换页面、不复制粘贴，在输入框内直接完成翻译到发送的全流程

### 1.4 目标用户
| 优先级 | 用户群体 | 使用场景 |
|---|---|---|
| 🥇 首要 | 跨境电商客服 / 外贸销售 | WhatsApp Web、Gmail 客户沟通 |
| 🥈 次要 | 平台后台 / 独立站客服 | 常见网页表单、站内信输入框 |
| 🥉 潜在 | 英文办公个人用户 | 日常英文邮件、表单填写 |

### 1.5 MVP 范围

第一版 MVP 聚焦跨境电商和外贸沟通的高频输入场景：

- **核心平台：WhatsApp Web、Gmail**
- **兜底能力：通用 textarea 与常见 contenteditable 输入框**
- **暂不承诺稳定适配：Outlook Web、Twitter/X、Facebook Messenger、LinkedIn 等复杂站点**
- **免费版不读取上下文**，仅处理用户当前输入框内主动输入的中文内容

## 二、核心功能需求

### 2.1 触发机制

| 项目 | 说明 |
|---|---|
| 触发方式 | 在任意输入框内，**连续按3次空格键** |
| 触发条件 | 输入框内有中文内容（不为空） |
| 触发结果 | 输入框下方弹出候选框，原输入内容暂存 |
| 备用快捷键 | `Ctrl + Shift + E` 同样触发翻译 |

### 2.2 候选框展示

触发后，在当前输入框正下方浮层展示候选框，包含以下信息：

**顶部状态栏**
- 显示当前翻译方向（中文 → English）
- 显示当前场景（如：跨境客服）
- 显示当前语气（如：礼貌安抚）

**3条候选句**，每条包含：
- 编号（1 / 2 / 3）
- 表达标签（如：礼貌自然 / 简洁直接 / 更正式）
- 对应英文译文完整展示

**示例（基于"很抱歉，物流延迟两天"）：**
```
1  礼貌客服   We're sorry for the delay. The shipment may be delayed 
              by about two days due to weather conditions, and we'll 
              continue to follow up on it for you.

2  简洁直接   Sorry for the delay. Your order may arrive two days 
              later due to the weather. We'll keep tracking it.

3  更正式     We sincerely apologize for the inconvenience. Due to 
              weather conditions, your shipment may be delayed by 
              approximately two days.
```

### 2.3 候选选择方式

| 操作 | 效果 |
|---|---|
| 键盘按 `1` / `2` / `3` | 对应候选填入输入框，候选框关闭 |
| 鼠标点击任意候选行 | 同上 |
| 按 `Esc` | 关闭候选框，恢复原始中文内容 |

### 2.4 填入与发送

- 选定候选后，英文内容**替换**输入框内的中文原文
- 插件**不自动发送**，光标停留在输入框内
- 用户检查内容后，自行按 `Enter` 或点击发送按钮

## 三、插件 Popup 面板需求

用户点击浏览器工具栏插件图标后展开面板，包含以下模块：

### 3.1 当前状态显示
- 输入语言（默认：中文）
- 输出语言（默认：English）
- 当前默认场景（如：跨境客服）
- 当前默认语气（如：礼貌自然）

### 3.2 快捷设置入口
- 「切换场景」按钮
- 「切换语气」按钮
- 「术语库」入口（Pro 功能）

### 3.3 用量显示（免费版）
- 今日已用次数 / 每日上限（如：18 / 30 次）
- Pro 功能状态（未开启 / 已开启）
- 升级 Pro 引导按钮

### 3.4 快捷键说明
```
Ctrl + Shift + E    翻译当前句
Tab                 使用推荐候选（选项1）
Esc                 关闭候选框
```

### 3.5 底部导航
- 打开设置
- 查看历史
- 反馈问题

## 四、场景与语气系统

### 4.1 场景列表（初期支持）

| 场景 | 适用平台举例 |
|---|---|
| 跨境客服（默认） | WhatsApp、亚马逊消息、独立站客服 |
| 外贸邮件 | Gmail、Outlook |
| 社媒互动 | Twitter/X、Facebook、LinkedIn |
| 通用翻译 | 所有其他输入框 |

### 4.2 语气列表（初期支持）

| 语气 | 说明 |
|---|---|
| 礼貌自然（默认） | 客服标准友好语气 |
| 简洁直接 | 去除客套，快速传达信息 |
| 正式专业 | 适合商务邮件、B2B 沟通 |
| 安抚道歉 | 处理投诉、退款、差评场景 |

> 场景和语气的组合会作为 Prompt 参数传入后端 AI 接口，影响生成结果风格。

## 五、适配平台范围（优先级）

| 优先级 | 平台 | 输入框类型 |
|---|---|---|
| P0（必须） | WhatsApp Web | contenteditable div，MVP 核心验收平台 |
| P0（必须） | Gmail | 富文本编辑器，MVP 核心验收平台 |
| P0（必须） | 通用 textarea | 标准 HTML textarea，兜底能力 |

| P2（后期） | Outlook Web | 富文本编辑器 |
| P2（后期） | Twitter/X、Facebook Messenger、LinkedIn | contenteditable div |

## 六、后端 & AI 接口需求

### 6.1 架构说明
- 插件不直接调用 AI API
- 所有翻译请求通过**开发者自建后端**中转
- 后端持有 AI API Key，用户不可见

### 6.2 请求参数（插件 → 后端）

| 参数 | 类型 | 说明 |
|---|---|---|
| `text` | string | 用户输入的中文原文 |
| `scene` | string | 当前场景（如 `cross_border_cs`） |
| `tone` | string | 当前语气（如 `polite`） |
| `user_id` | string | 用户唯一标识（用于额度控制） |

### 6.3 返回结果（后端 → 插件）

| 参数 | 类型 | 说明 |
|---|---|---|
| `candidates` | array | 3条候选句，每条含 `label`（语气标签）和 `text`（英文内容） |
| `remaining` | number | 用户今日剩余次数 |
| `error` | string | 错误信息（如超额、服务异常） |

### 6.4 AI 模型要求
- 使用具备自然语言生成能力的大语言模型（非传统翻译 API）
- 需支持通过 Prompt 控制输出语气和场景风格
- 每次请求返回 **3条风格不同的候选句**

## 七、用量与权限控制

### 7.1 免费版限制
- 每日翻译上限：**15 次**（基于 GPT-4o mini 成本测算：单次约 ¥0.00114，15次/天约 ¥0.017/用户/天，1000免费用户每月成本约 ¥513，可控）
- 候选数量：3条（不区分语气标签）
- 场景支持：仅通用翻译 + 跨境客服
- 术语库：不支持
- 历史记录：最近 10 条
- 上下文读取：不支持，仅翻译当前输入框内容

### 7.2 Pro 版权益（后期实现）
- 每月翻译额度大幅提升
- 支持全部场景与语气
- 多候选语气标签细化
- 个人术语库
- 历史记录无上限
- 上下文辅助回复：可读取用户主动授权或手动提供的聊天 / 邮件上下文

## 八、非功能性需求

| 项目 | 要求 |
|---|---|
| 响应速度 | 触发后候选框展示时间 < 3秒（含 AI 生成） |
| 候选框位置 | 跟随输入框，不超出视口边界 |
| 最小侵入性 | 不主动弹窗、不影响网页原有功能，不自动发送消息 |
| 数据安全 | 免费版仅处理当前输入框内容，用户输入内容仅用于单次翻译，不持久化存储 |
| 浏览器兼容 | Chrome 最新版（Manifest V3） |

## 九、暂不在本期实现的功能

- 团队版 / 团队话术库
- 客服质量检查
- 自动上下文感知回复（读取对方上一条消息，后续 Pro 功能）
- 订阅支付流程
- 批量改写
- 企业版 / 私有化部署
